## Summary: 4 transitive deps fail to compile

| Crate | Errors | Root cause |
|---|---|---|
| `serde_core` (1.0.228) | 5825 | `extern crate alloc` / `use core::prelude` not in scope. `serde = { default-features = false, features = ["derive", "alloc"] }` was set but `serde_core` is a NEW split crate as of `serde 1.0.228` and may need its own `alloc` feature gate, or upstream needs a fix. **Transitive via `ciborium → serde`** — and `ciborium` is currently NOT no_std-friendly per #93's hold-back. May resolve itself once `ciborium`/`ciborium-ll` are gated off the verifier path entirely. |
| `byteorder` (1.5.0) | 91 | Not flipped to `default-features = false` in #93 (Codec trait still uses `ReadBytesExt`/`WriteBytesExt`). 91 errors all from byteorder's no_std arm having stale `std::*` references. **Resolution: `default-features = false` + rewrite the Codec trait's `read_u24`/`write_u24` calls** to use slice-direct LE operations. This is the centerpiece of #94. |
| `ciborium-io` | 9 | Pulled transitively via `ciborium`. `ciborium-io` depends on `std::io::{Read, Write}`. **Resolution: gate `ciborium`/`ciborium-ll` so they only compile under `feature = "prover"`** — verifier path doesn't need CBOR encode/decode if the proof's CBOR-typed fields can be passed in/out as already-extracted bytes. May require a small refactor in `crates/longfellow/src/mdoc_zk/mdoc/cose.rs` and friends. |
| `const-oid` (transitive via `x509-cert`) | 1 | One `std::*` reference. Probably fine after enabling its no_std feature, but `x509-cert = { features = ["hazmat", "std"] }` currently includes `std` explicitly. **Resolution: drop `std` from `x509-cert`'s feature list** or gate `x509-cert` to prover path. |

## Prep work landed in #93 / 1.2b1 (already in tree)

- `crates/longfellow/src/io.rs` — slice-cursor + Vec-Write shim, ready for
  cascade.
- `crates/longfellow/Cargo.toml` — `default-features = false` on
  `hex`, `num-bigint`, `num-integer`, `serde`, `sha2`, `subtle`. Hold-backs
  documented inline.
- `lib.rs` has `extern crate alloc;` (so the no_std arm of `crate::io`
  resolves) but **no** `#![cfg_attr(not(feature = "std"), no_std)]` activation
  — that flip is the FIRST step of #94.
- Prover modules (`mdoc_zk`, `zk_one_circuit`, `ligero`) all gated
  `#[cfg(feature = "prover")]` from the earlier 1.2b1 step 1 commit.

## Suggested #94 sub-step ordering

1. **Decide ciborium/x509-cert fate.** Either gate them to `feature = "prover"`
   (cleanest if the verifier doesn't decode CBOR/X.509 directly) or find a
   no_std-friendly subset. Read the verifier path in
   `crates/longfellow/src/mdoc_zk/{verifier.rs,mdoc/cose.rs,mdoc/mod.rs}` to
   confirm what CBOR/X.509 surface area survives feature-gating.
2. **Flip `byteorder` to `default-features = false`.** Rewrite the `Codec`
   trait body in `lib.rs` (~50 LoC) to use `LittleEndian::read_*`/`write_*`
   slice operations directly, plus add `read_u24/write_u24` methods to
   `crate::io::Cursor` and the `Write` shim trait. This single change unblocks
   all 91 byteorder errors AND every callsite that uses `cursor.read_u32::<…>()`.
3. **Activate `#![cfg_attr(not(feature = "std"), no_std)]`** in lib.rs.
4. **27-file `std::*` → `core::*` / `alloc::*` / `crate::io::*` cascade**
   per the audit list in #93's brief.
5. **Re-run riscv32 check.** Expect a new wave of crate-internal errors —
   each one is now small and addressable.
6. **Document feature matrix** in `crates/longfellow/README.md`.

## Hold-backs from 1.2b1 (re-flip in 1.2b2)

These were reverted to `default-features` because flipping them broke the
HOST `--no-default-features --features verifier` build. Each is a forward
todo for #94:

- `aes` → flipping cascades into crypto-common losing `std` ⇒ flips
  `InvalidLength: std::error::Error` off ⇒ anyhow::Context breaks. Resolution
  in #94: stop using `anyhow::Context` on `Aes256::new_from_slice(...)`'s
  `Result<_, InvalidLength>` (manual map_err to `anyhow!("bad key length")`).
- `byteorder` → see step 2 above.
- `crypto-common` → currently has explicit `features = ["std"]` to preserve
  the impl. Drop after the anyhow::Context callsites are rewritten.
- `educe` → flipping drops the `PartialEq` derive. Resolution: derive
  `PartialEq` manually on `SparseSumcheckArray<FE>` once educe goes no_std.
- `ciborium`/`ciborium-ll` → gate to `feature = "prover"` (see step 1).
